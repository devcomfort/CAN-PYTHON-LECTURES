# 자료형 개요

- 자료형은 `저장할 수 있는 변수의 형태`
- 크게 `문자`, `정수`, `실수`로 나뉨
  - `정수` : 음수(-), 0, 양수(+)
  - `실수` : `정수`에 소수점 붙은 숫자 (예시: 3.14)
- 콜렉션(collection), 이터러블(iterable)이라는 개념도 있음
  (리스트, 딕셔너리, 튜플 등)

---

# 자료형 개요

- 숫자 : int, float, complex
- 문자 : str
- 논리 : bool
- 콜렉션/이터러블 : list, dict, set, tuple

---

# 숫자 자료형

```python
a: int = 10  # 정수
b: float = 3.14  # 실수
c: complex = 3 + 11j  # 허수
```

# 문자 자료형

```python
string: str = "Hello World!"  # 문자 데이터
char: str = "A"  # 파이썬에서는 문자(character) === 문자열(string)
```

---

# 논리 자료형

논리값은 `True`와 `False`로 구분됨 (깊게 가면 Truthy와 Falsy라는 개념도 있음)

```python
true: bool = True  # 참 값
false: bool = False  # 거짓 값
```

# 논리 자료형

false: bool = True  # 참 값
true: bool = False  # 거짓 값

# 리스트 자료형

값을 여러 개 저장할 수 있는 자료형. 값의 자료형과 상관 없이 다양한 형태 저장 가능

```python
리스트: list = [1, "2", "Hello World!", [1, 2, 3]]  # 다른 리스트, 정수, 문자 등 다 저장할 수 있음
리스트[0]  # 값에 접근하는 법 : 0부터 시작하여 순서대로 접근 가능
리스트[3][0]  # 리스트 내부의 값이 리스트라면 이렇게도 가능 : [1, 2, 3]의 [0]번째 -> 1

# (나름 고급 개념) 리스트[a:b] -> a번째 ~ b-1번째 요소를 가져와서 새로운 리스트 생성
리스트[1:3]  # 리스트의 1, 2 번째 요소를 가져와서 새로운 리스트 생성 -> ["2", "Hello World!"], '슬라이싱' 개념

# (고급 개념) asterisk operator
[0, 1, *리스트[0:2]]  # -> [0, 1, 1, "2"]
```

---

# 튜플 자료형

값을 여러 개 저장할 수 있지만, 값을 변경할 수 없는 자료형 (immutable list?)
쉽게: '값 못 바꾸는 리스트'

```python
튜플: tuple = (1, "2", "Hello World!", [1, 2, 3])  # 튜플도 다른 리스트, 정수, 문자 등 다 저장할 수 있음
튜플[0]  # 1
튜플[3][0]  # -> 1
튜플[1:3]  # -> ("2", "Hello World!")

(0, 1, *튜플[0:2])  # -> (0, 1, 1, "2")
```

---

# 집합 자료형

리스트, 튜플과는 다르게 '중복 불가'에 초점을 맞춘 자료형.
여러 개의 값을 저장할 수는 있지만, 한 번에 하나의 값을 저장하는 자료형만 저장 가능

```python
집합: set = {1, 2, 3}
집합.add([1, 2, 3])  # 불가. 해싱 가능한 자료형만 가능 = 한 번에 하나의 값을 저장하는 자료형만 가능
```

# 딕셔너리 자료형 (조금 어려움)

데이터를 key-value pair 형태로 저장하는 자료형

key-value pair는 '값에 접근하기 위한 열쇠 값'과 '열쇠로 접근할 수 있는 값 그 자체'를 한 묶음으로 저장하는 자료형

```python
딕셔너리: dict = {
  "key": "value",
  "A": "I'm A",
  "B": "I'm C"
}

딕셔너리["key"]  # "value"
딕셔너리["A"]  # "I'm A"
딕셔너리["B"]  # "I'm C"

# 아래와 같은 형태로 일반화 가능
# 딕셔너리[key] -> value
```
---

# 딕셔너리 자료형 (더 어려운 거)

딕셔너리 자료형에서의 asterisk operator 사용법.
asterisk operator(\* 연산자)는 함수, '가변 인자'에서 또 나올 예정입니다.

```python
# asterisk operator는 * 또는 ** 형태로 "데이터를 풀어내는데" 사용됨
a: list = [1, 2, 3]
b: list = [-1, -2, -3]
[*b, *a]  # [-1, -2, -3, 1, 2, 3] <- 이런 식


# 딕셔너리 자료형과 리스트/튜플 자료형에서의 asterisk operator의 사용법이 다름
# 딕셔너리에서는 이렇게:
a: dict = { "A": "I'm A" }
b: dict = { "B": "I'm B" }
{**a, **b}  # 이렇게 하면 { "A": "I'm A", "B": "I'm B"} 와 같은 자료형이 됨
[*a, *b]  # 이렇게 하면 key만 가져올 수도 있음 -> ["A", "B"]
```

# 연산자
